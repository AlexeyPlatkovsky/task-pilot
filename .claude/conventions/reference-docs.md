# Reference Documentation Standard

- `.claude/docs/` stores reusable AI-facing project knowledge; `docs/` stores product specifications,
  decisions, concepts, and user/developer documentation.
- Maintain `.claude/docs/README.md` as the lookup index.
- Read the smallest relevant file or stable section first and expand only when necessary.
- Keep reference docs factual. They must not contain routing, execution procedures, validation
  gates, or agent responsibilities.
- Update existing authoritative sources narrowly and link rather than duplicate facts.
- Use purposeful subfolders when multiple domains or document families emerge.
