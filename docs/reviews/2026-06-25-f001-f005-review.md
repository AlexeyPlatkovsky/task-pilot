## 5. Findings

### Quality gate — needs action

- **F-1 (Minor): Format drift.** `ruff format --check` would reformat 5 files:
  `src/taskpilot/cli/commands/serve.py`, `src/taskpilot/server/routes/projects.py`,
  `src/taskpilot/server/schemas.py`, `tests/server/test_api.py`,
  `tests/services/test_item_service.py`. **Fix:** run `ruff format .`.

### Architecture

- **F-2 (Major, accepted): Adapter→adapter import.** `cli/commands/serve.py:19` imports
  `taskpilot.server.app.create_app` (CLI adapter → REST adapter). Normally a boundary violation, but
  explicitly accepted in `docs/decisions/ADR-006-system-registry-in-cli.md` and tracked as **TP-4**
  for extraction to a service factory. Action: keep TP-4 open and scheduled.

### Low / cosmetic (from independent review)

- **F-3 (Low): `findings: null` vs optional TS field.** Valid `ItemSummary` serializes
  `findings: null` (`server/schemas.py:38`); TS models it as optional (`web/src/types/index.ts:63`).
  Interoperable; consider defaulting to `[]` or `findings?: ... | null`.
- **F-4 (Low): Item endpoints not scoped to path `project_id`.** `routes/projects.py:110-131` resolve
  the workspace from `project_id` but do not assert the item belongs to that project key. Benign while
  one repo holds one project; matters if multi-project-per-repo is introduced.
- **F-5 (Low): Empty PATCH still rewrites file / bumps `updated_at`.** A no-op `PATCH {}` (or CLI
  `item update` with no options) mutates `updated_at` and produces a Git diff. Consider short-circuit
  on empty field set.
- **F-6 (Nit): Comment body newline normalization is asymmetric** between parse and dump; the
  byte-stable file round-trip invariant still holds.

### Documentation drift (observed)

- `docs/INDEX.md` Feature Registry lists **F001 = planned** and **F003 = in progress**, but both are
  implemented and fully passing tests. Consider updating the registry statuses. (Not changed here —
  flagged for a separate doc-maintenance pass.)

## 6. Invariants Confirmed

- Markdown/YAML files are canonical; deterministic serialization (`sort_keys=False`, canonical field
  order); canonical-before-index; atomic writes throughout.
- Reverse links derived, not stored; links rewrite only the source file.
- Invalid files remain visible (loader/validation fault-tolerant; API surfaces unparseable items as
  `valid:false`).
- Offline/local-first; path-traversal guards in `layout` and attachment validation.
- core/services contain no upward or adapter imports (only accepted exception is F-2).

## 7. Recommendations

2. Keep TP-4 tracked; schedule the `serve.py` → service-factory extraction (F-2).
3. Address F-3..F-5 opportunistically (low risk).
4. Update `docs/INDEX.md` feature statuses for F001/F003 to reflect implemented state.

## 9. Resolution (2026-06-26)
